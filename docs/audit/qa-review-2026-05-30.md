## 2. Findings

### W1 — Nit — Design

**`lib/disguises.ts` — `DisguiseId` union type will require code changes to add new disguises**

```typescript
export type DisguiseId = 'hipaa' | 'pci' | 'glba' | 'gdpr' | 'ccpa' | 'ferpa' | 'sox' | 'default'
```

The `DisguiseId` type is a string-literal union manually kept in sync with the `DISGUISES` array. Adding a new regulation requires updating the union, the array, and any exhaustive switch statements referencing the type. A derived type would be safer:

```typescript
export const DISGUISE_IDS = ['hipaa', 'pci', 'glba', 'gdpr', 'ccpa', 'ferpa', 'sox', 'default'] as const
export type DisguiseId = (typeof DISGUISE_IDS)[number]
```

This makes the union the single source of truth and ensures any exhaustive switch statements catch new entries at compile time.

---

### W2 — Nit — Data Accuracy

**`lib/disguises.ts` HIPAA sample row — `mrn` output is 12 hex chars (6 bytes); should document the truncation unit**

```typescript
{ column: 'mrn', input: 'MR-20847', output: '8a4f2c1b3d5e' },
```

The identifier table says "hash sha256, truncated to 12 chars". The sample output `8a4f2c1b3d5e` is 12 hexadecimal characters (6 bytes). This is consistent, but the YAML sample says `truncate: 12` which could be interpreted as 12 bytes (24 hex chars) or 12 characters. The documentation would benefit from clarifying that `truncate: 12` means 12 hex-output characters (6 bytes of hash), not 12 bytes. Add a `notes:` field to the YAML sample:

```yaml
  - match:
      detectors: ["mrn"]
    mask: hash
    params: { algorithm: sha256, truncate: 12 }
    notes: "truncate: 12 means 12 hex output characters (6 bytes of hash output)."
```

## 5. What's Good

The **honesty sections** in every Disguise are the standout design choice: each Disguise explicitly documents what it does NOT cover (Expert Determination for HIPAA, reversible tokenisation for PCI, consent management for GDPR, etc.). This is the right UX for a compliance-adjacent tool — setting accurate expectations prevents misuse.

The **`triggerDescription` and `triggers` YAML** for each Disguise correctly model co-occurrence-based detection (e.g., HIPAA fires on `date + ZIP + name` together, not just any date alone), which matches the STORM scanner's quasi-identifier detection model.

The **discriminated union pattern** (`DisguiseId` + `DISGUISES: Disguise[]`) allows exhaustive TypeScript handling of all regulation variants at compile time.
